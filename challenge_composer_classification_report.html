Data Challenge - Part1: NLP-Based Composer Classification¶

Author: Emmanuel Ekwedike

Date: 08/18/23

Abstract¶

In this work, we build a composer classifier based on live captured audio, streaming data to our model every 30 seconds. For this project, Our dataset consists of numerous annotated audio files saved in MIDI format. To represent each musical piece within the MIDI files, we've employed an NLP-based approach. Using the extracted features, we constructed three machine-learning algorithms. Our experiments underscore the model's sensitivity to both feature extraction and data quality. With the present data, our models achieved an impressive F1-score of 0.97 during cross-validation.

Introduction¶

Music, a universal language, transcends boundaries and evokes emotions. In the realm of technology and creativity, the fusion of music and machine learning opens doors to new possibilities.

The literature has witnessed endeavors to identify music styles and composers through diverse computational methods. The emergence of statistical machine learning, deep learning, and statistical NLP has been pivotal in unraveling insights into music representation [Wołkowicz2008].

Music's similarity to natural language has opened avenues for employing NLP tools in music analysis. Drawing inspiration from recent successes in NLP-based approaches [Wołkowicz2008], Deepaisarn2023a], we adapted a similar methodology proposed by [Deepaisarn2023a].

The rest of the report is organized as follows: Methodology, Results, Discussion, Conclusion, and Recommendation.

Dataset¶

This study utilized annotated musical data extracted from Musicnet [Thickstun2016], presented in MIDI format. The data comprises MIDI files associated with four composers: Bach, Beethoven, Schubert, and Brahms. These MIDI files encompass extra details, like composition names and instruments, which might not be accessible during inference. Consequently, we'll exclude this supplementary information during algorithm development.

def extract_Pi_Di_tuple(df_metadata, use_saved_data = False, save_path= "./all_gram_list_note.npy"):
    """
        :param df_metadata:
        :param use_save:
        :param save_path:
        :output all_gram_list, df_metadata_subset: list of all cordinate pairs, and subset of metadata 
    """
    if use_saved_data:
        all_gram_list = np.load(save_path)
        corrupt_indices = []
    else:
        all_gram_list = []
        cnt = 0
        corrupt_indices = []
        for i in range(df_metadata.shape[0]):
            suffix_path = df_metadata["midi_filename"][i]
            path = os.path.join(base_path, suffix_path)
            try:
                frame = du.convert_midifiles_to_notes(path)
                gram_list = du.extract_gram(frame)
                all_gram_list = all_gram_list + gram_list
            except:
                corrupt_indices.append(i)
                continue
        np.save(save_path,all_gram_list)
        all_gram_list = np.array(all_gram_list)

    df_metadata_subset = df_metadata.drop(index = corrupt_indices).copy()
    df_metadata_subset.reset_index(inplace=True)
    return all_gram_list, df_metadata_subset

def build_corpus(df_metadata_subset, all_gram_list, filename = "corpus.txt"):
    """
        :param df_metadata_subset:
        :param all_gram_list:
        :param filename: 
        :output : None
    """
    sorted_gram_list = sorted(set(tuple(i) for i in all_gram_list.tolist()))
    note2Char = { j:du.encodeChinese(i) for i,j in enumerate(sorted_gram_list)}
    Char2note =  { du.encodeChinese(i):j for i,j in enumerate(sorted_gram_list)}

    corpus_text = ''
    for i in range(df_metadata_subset.shape[0]):
        suffix_path = df_metadata_subset["midi_filename"][i]
        path = os.path.join(base_path, suffix_path)
        frame = du.convert_midifiles_to_notes(path)
        gram_list = du.extract_gram(frame)
        for j in gram_list:
          corpus_text += note2Char[j]
        corpus_text += '\n'
    with open(filename, "w") as f:
        f.write(corpus_text)

def sentencePiece(corpus, modelName, vocabSize, maxSenLength):
    """
        :param corpus:
        :param modelName:
        :param vocabSize: 
        :param maxSenLength:
        :output : sentences
    """
    MAXITER =1276
    spm.SentencePieceTrainer.train(input=corpus, model_prefix=modelName, vocab_size=vocabSize, max_sentence_length=maxSenLength)
    sp = spm.SentencePieceProcessor()
    temp = modelName+".model"
    sp.load(temp)
    f1 = open(corpus,'r')
    temp = {}
    for i in range(MAXITER):
        line = f1.readline()
        
        tokenized = sp.encode_as_pieces(line)
        temp[i] = tokenized
    Ch_note_series = pd.Series(temp)
    f1.close()
    sentences = []
    for i in range(Ch_note_series.shape[0]):
        sentences.append(Ch_note_series[i])
    return sentences

def get_metric_acc(sample_indices, X, y):
    """
    Calculate performance metrics for classification using different classifiers.

    This function calculates classification metrics such as F1-score for multiple classifiers (K-Nearest Neighbors,
    Random Forest, and Logistic Regression) based on the provided sample indices and data.

    Parameters:
    - sample_indices (list or array): Indices of the samples to use for training the classifiers.
    - X (array-like): Feature matrix of the dataset.
    - y (array-like): Target labels of the dataset.

    Returns:
    - knn_f1_score (float): Weighted F1-score for the K-Nearest Neighbors classifier.
    - rf_f1_score (float): Weighted F1-score for the Random Forest classifier.
    - lr_f1_score (float): Weighted F1-score for the Logistic Regression classifier.
    """
    mask = np.zeros((X.shape[0],1),dtype=bool)
    mask[tuple(sample_indices),:] = True
    y_sub = y.reshape(-1,1)[mask]
    # y_sub = y.reshape(-1,1)[tuple(sample_indices),:]
    y_sub = y_sub.reshape(-1)
    y_sub_test =y.reshape(-1,1)[~mask]
    y_sub_test = y_sub_test.reshape(-1)

    X_sub  = X[mask.reshape(-1)]
    X_sub_test = X[~ mask.reshape(-1)]

    clf_knn = KNeighborsClassifier(n_neighbors=4)
    knn = clf_knn.fit(X_sub,y_sub)
    
    clf_rf = RandomForestClassifier(max_depth=500, random_state=0)
    rf = clf_rf.fit(X_sub,y_sub)
    
    clf_lr = LogisticRegression(random_state=0, max_iter=300)
    lr = clf_lr.fit(X_sub,y_sub)
    
    knn_pred=knn.predict(X_sub_test)
    rf_pred=rf.predict(X_sub_test)
    lr_pred=lr.predict(X_sub_test)
    # knn_acc = accuracy_score(y_sub_test, knn_pred)
    # rf_acc = accuracy_score(y_sub_test, rf_pred)
    # lr_acc = accuracy_score(y_sub_test, lr_pred)
    knn_acc= metrics.f1_score(y_sub_test, knn_pred, average='weighted')
    rf_acc= metrics.f1_score(y_sub_test, rf_pred, average='weighted')
    lr_acc= metrics.f1_score(y_sub_test, lr_pred, average='weighted')
    # print(f"knn: {knn_acc}, rf: {rf_acc} and lr: {lr_acc}")
    return knn_acc, rf_acc, lr_acc

Machine Learning Classification Models¶

In the following section, we outline the implementation of three classical machine learning models: logistic regression (LR), random forest (RF), and k-nearest neighbors (KNN). These models are employed to classify music samples into four distinct categories, each corresponding to composers Bach, Beethoven, Schubert, and Brahms.

To facilitate experimentation, the processed dataset comprising 184 samples is partitioned into training and testing subsets. The training set is utilized for model training, while the testing set serves for model evaluation. The division ratio between train and test sets is set at 80% and 20% respectively.

To assess the performance of the classifiers, we employ the weighted F1 score as our chosen metric. This selection is motivated by the multi-class nature of the classification task and the need to address data imbalances observed within the dataset. Furthermore, cross-validation results are documented to ensure the reliability and robustness of our chosen evaluation metric.

def run_random_cross_validation(classifier, X, y, NUM_CV_TRIAL = 100, holdoutratio = 0.2):
    """
    Run random cross-validation for a classifier.

    This function performs random cross-validation for a given classifier on a dataset, assessing its performance
    over multiple trials. It randomly splits the data into training and validation sets for each trial and reports
    evaluation metrics.

    Parameters:
    - classifier: The classifier to be evaluated.
    - X: The feature matrix of the dataset.
    - y: The corresponding target labels.
    - NUM_CV_TRIAL (int): The number of cross-validation trials to run (default is 100).
    - holdout_ratio (float): The ratio of data to hold out for validation in each trial (default is 0.2).

    Returns:
    - mean_accuracy (float): The mean accuracy over all cross-validation trials.
    - mean_precision (float): The mean precision over all cross-validation trials.
    - mean_recall (float): The mean recall over all cross-validation trials.
    - mean_f1_score (float): The mean F1 score over all cross-validation trials.
    """
    print(f"Running Cross validation for model {classifier.__class__.__name__}")
    list_f1_scores = []
    for _ in range(NUM_CV_TRIAL):
        ind = np.arange(len(y))
        np.random.shuffle(ind)
        split = round(len(y) * (1-holdoutratio))
        X_train = X[ind[:split]]
        X_test = X[ind[split:]]
        y_train = y[ind[:split]]
        y_test = y[ind[split:]]
        cls=classifier.fit(X_train,y_train)
        prediction=cls.predict(X_test)
        f1_Score_cls=metrics.f1_score(y_test, prediction, average='weighted')
        list_f1_scores.append(f1_Score_cls)
    avg_f1_score = np.mean(list_f1_scores)
    std_f1_score = np.std(list_f1_scores)
    return avg_f1_score, std_f1_score

Discussion¶

We exclusively examined traditional machine learning approach(es) and established a comprehensive pipeline for those methods. Through this, we achieved a remarkable cross-validation result – a weighted F1 score of 0.98 on imbalanced data. Additionally, there is minimal variance among various cross-validation samples. This outcome underscores robust performance.

Classifier Performance: The weighted F1 score is a measure that considers both precision and recall, making it a suitable metric for imbalanced datasets. An F1 score of 0.98 indicates that the classifier is able to effectively balance between precision and recall across all classes, resulting in very accurate predictions.

Weighted F1 Score: The weighted F1 score takes into account the class distribution while calculating the overall F1 score. This is crucial when dealing with imbalanced data because a simple average of F1 scores might not provide an accurate representation of the classifier's performance. The weighted F1 score assigns more weight to classes with fewer samples, giving a better insight into how well the classifier handles the minority classes.

Balancing Precision and Recall: F1 score is the harmonic mean of precision and recall. Precision measures the ratio of true positive predictions to the total predicted positives, while recall measures the ratio of true positive predictions to the total actual positives. Achieving a high F1 score indicates that the classifier is successfully balancing the trade-off between making accurate positive predictions (precision) and capturing the actual positives (recall).

Business Impact: The strong classifier's performance can bring substantial benefits. For example, in the client's application of identifying compositions beyond the four trained composers, accuracy in classification is crucial. This classifier has the potential to enhance decision-making reliability and accuracy.

def detect_unknown_composer(classifer, X_validation,threshold = 0.55):
    """
    Detect unknown composers using a classifier.

    This function takes a pre-trained classifier and a set of validation data (X_validation) to predict whether the composers
    of the given data are unknown based on a specified threshold.

    Parameters:
    - classifier: A pre-trained composer classifier model.
    - X_validation: Validation data containing features for composer classification.
    - threshold (float): The threshold for deciding if a composer is unknown (default is 0.55).

    Returns:
    - unknown_composers (list): A list of indices corresponding to the data samples with predicted unknown composers.
    - predicted_probabilities (list): A list of predicted probabilities for each data sample.
    """
    val_probabilities = classifer.predict_proba(X_validation)
    print(f"Probabilities of the validation data\n {val_probabilities}")
    
    # Identify samples not belonging to any class
    index_samples_not_belonging = []
    for i, probabilities in enumerate(val_probabilities):
        if all(p < threshold for p in probabilities):
            index_samples_not_belonging.append(i)
    
    # Print the indices of samples not belonging to any class
    len_of_outliers = len(index_samples_not_belonging)
    print(f"At threshold {threshold}, the number of sample(s) not belonging to any of the 4 classes: {len_of_outliers}")
    return index_samples_not_belonging

Conclusion and Recommendations¶

In this study, we develop a composer classifier using real-time audio inputs. Our dataset comprises annotated MIDI format audio files. We utilize an NLP-based method for feature extraction, yielding promising results. Our models achieve a notable F1-score of 0.97 in cross-validation experiments.

Recommendations¶

Our primary recommendation centers on ensuring robustness.

  1. Data Collection Stage Robustness: Enhance data collection with a focus on both training and validation data. Enriching the validation set will improve model evaluation accuracy and aid in refining detection thresholds for out-of-sample composers. The training process will benefit from additional data, especially if challenging negative samples are collected to supplement the existing data. Incorporating difficult data and adversarial examples will significantly enhance model robustness. Additionally, address data imbalance during collection to ensure effective learning across all classes.

  2. Evaluation Process Robustness: Reassess model selection in light of augmented data. Ensemble and deep learning methods offer potential improvements over existing techniques. Consider advanced feature engineering to extract more generalized audio features, thereby enhancing overall evaluation. Regularly enhance the model by monitoring performance, gathering deployment environment feedback, and analyzing how temporal and environmental changes impact model effectiveness.
